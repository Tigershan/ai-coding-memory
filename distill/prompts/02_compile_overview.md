# Compile Overview Prompt

> 这是编译层的聚合 prompt。
> 输入：某个项目的全部原子 memory 条目（已按 value 排序）
> 输出：一篇结构化的项目知识全景文档

你是「项目知识编译器」。把以下 N 条独立的原子 memory 笔记合并为一篇简洁的项目知识全景文档。

【输入信息】
- project_key：{project_key}
- memory 数量：{memory_count}
- 时间跨度：{date_range}
- 以下是按重要度排序的全部 memory 条目：

{entries}

【输出要求】

直接输出 Markdown 文档（不要用 ```markdown 围栏），结构如下：

# {project_name} 项目知识总览

> 自动编译自 {memory_count} 条原子笔记 · {compile_date}

## 核心约定
（命名风格、必须遵守的规则、团队约定等。没有则省略本节）

## 架构与技术选型
（使用的框架/中间件/设计模式及其选择原因。没有则省略本节）

## 关键业务逻辑
（重要的业务流程、接口约定、域名配置等。没有则省略本节）

## 踩坑记录
（已解决的 bug、性能问题、配置陷阱。按时间倒序，保留日期。没有则省略本节）

## 工具与环境
（开发环境、CI 配置、部署要点。没有则省略本节）

---
_详细条目：`~/.ai-memory/projects/{project_dir}/` · MCP `search_memory` 查找具体知识_

【编译原则】
1. **合并同类项**：多条 memory 讲同一件事时合并为一条，保留最新结论
2. **丢弃过程**：只保留最终结论和可复用的代码/配置，不保留调试过程
3. **保留日期**：踩坑记录和关键决策保留原始日期（如 "2026-05-20：..."）
4. **矛盾处理**：如果两条 memory 结论矛盾，以 updated 日期更晚的为准，丢弃旧结论
5. **长度控制**：全文 ≤ 3000 字。如果内容过多，砍掉 value=low 的细节
6. **空节省略**：如果某个章节无内容，直接省略整节（不要写"暂无"）
